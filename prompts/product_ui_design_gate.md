# Step 2B 产品化与 UI 设计门禁

## 目标

当 skill 需要协助开发最低可申请软件，或现有软件界面明显简陋时，必须先做产品化和 UI 设计判断。软著不审美术水平，但操作说明书和截图会直接暴露软件是否真实、完整、可信；粗糙界面会削弱“软件表达”和“实际应用场景”的说服力。

## 设计 brief

开发前生成 `草稿/UI设计方案.md`，至少回答：

- 目标用户：谁每天或在什么节点使用。
- 主任务：用户打开软件后的首要任务是什么。
- 信息架构：导航、工作区、结果区、日志/历史、设置/导出如何组织。
- 关键屏幕：不少于 3 个，通常为主界面、输入/配置、运行结果/报告。
- 状态设计：空状态、处理中、成功、失败、异常数据、无权限/无文件等。
- 视觉基调：工程工具、资料质检、科研分析、经营管理等不同场景应有不同密度和语气。
- 组件和技术栈：优先采用成熟 UI 框架/组件库，不手搓基础控件。
- 截图计划：最终说明书需要哪些完整窗口截图。

同时生成或更新 `截图/screenshot_manifest.json`，用于正式 Word 选择截图：

```json
{
  "screenshots": [
    {
      "file": "word_01_main_workspace.png",
      "order": 1,
      "flow_step": "主工作台",
      "viewport": "1440x900",
      "operation": "启动软件并进入主界面",
      "accepted": true,
      "used_in_word": true,
      "qa_note": "文字完整、控件未遮挡、能看出主任务入口"
    }
  ]
}
```

## 技术栈选择

- Web 应用优先：若软件需要表格、筛选、上传、报告、历史记录、图表或多页面流程，优先用 React/Vite/Next 等成熟前端栈，并配合 shadcn/ui、Material UI、Ant Design、Fluent UI 等组件库。
- 桌面应用：如必须本地桌面运行，优先 PySide6、Qt、Tauri、Electron 或 CustomTkinter；不要默认使用裸 Tkinter 做最终界面，除非用户明确要求或软件只是内部命令包装，并且必须记录原因和视觉 QA。
- 数据/工程工具：界面应安静、紧凑、面向扫描和复核；避免营销式 hero、大面积装饰渐变、空泛卡片堆叠。
- 图表/地图/流程图：使用成熟库，例如 ECharts、Recharts、Plotly、Leaflet、OpenLayers、Mermaid 渲染图片等。
- 图标：优先使用现有图标库，例如 lucide、Material Symbols、Fluent Icons；不要手绘临时 SVG 图标。

## UI 最低门槛

不得进入软著材料生成，除非满足：

- 首屏就是可用工作台，不是介绍页或说明页。
- 有清晰导航、主操作按钮、状态反馈、错误提示和结果区域。
- 关键任务不依赖用户记忆路径；按钮、字段、结果命名使用业务语言。
- 表格、列表、日志、导出、筛选、批量处理等控件符合用户任务。
- 所有主要操作有可截图状态：空状态、录入/选择、处理中、完成/异常。
- 文字不溢出、不乱码、不遮挡；截图在常见桌面分辨率下完整。
- 至少包含一个真实样例数据或演示流程，截图不是空界面。

## 设计 QA

开发后必须执行：

- 运行软件并采集完整窗口截图。
- 为每张候选截图写入 `screenshot_manifest.json`，只把 `accepted: true` 且 `used_in_word: true` 的截图交给 Word 生成阶段。
- 对 Web UI，用 Playwright 或浏览器检查桌面和移动/窄屏视口；对桌面 UI，用实际窗口截图检查完整性。
- 检查可访问性基本项：对比度、键盘焦点、表单标签、状态文本、错误恢复。
- 检查 Nielsen 启发式的核心项：系统状态可见、业务语言匹配、用户可撤销或重选、风格一致、错误预防、帮助与文档贴近任务。
- 检查正式说明书截图：不得使用裁切图、空白图、文字不完整图、IDE/聊天窗口图。

## 与软著材料的衔接

- UI 设计方案不是营销包装，它要服务于操作说明书、截图证据和软件表达真实性。
- 每个关键屏幕都应能映射到说明书章节和申请表“主要功能”。
- 若 UI 只能证明“文件转换”，不得在材料中包装成“全流程申报管理系统”。
- 若 UI 能证明“数据导入、规则质检、问题定位、结果导出、日志归档”，可以据此强化“申报质检工具”的业务联系。

设计方案确认后运行：

```bash
python tools/confirm_stage.py --workdir 软件著作权申请资料 --stage ui-design --note "<用户确认的 UI 方案、技术栈和截图计划>"
```
