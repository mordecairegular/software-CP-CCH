# 纠错处理

## 目标

当用户指出材料与事实不符、参数错误、功能不存在、版本错误或表达不合适时，立即纠错，不争辩，不扩大改动范围。

## 处理规则

1. 先定位错误出现在什么文件和章节。
2. 查找是否影响申请表、手册、代码材料、截图、文件名、页眉。
3. 若影响一致性，联动修正所有相关材料。
4. 若用户纠正的是项目事实，以用户新事实为准，并在修订记录中注明。
5. 若用户指出功能不存在、事实错误、材料太泛、申报对象不准或 UI/截图低质，必须回到 `software_concept_mining.md`、`business_understanding.md` 或 `product_ui_design_gate.md` 复核，不得只改几句文案。
6. 另存新版本，不覆盖旧稿，除非用户明确要求覆盖。

## 输出

- 纠错摘要。
- 修改文件列表。
- 仍需用户确认的问题。
- 更新后的自检结果。
