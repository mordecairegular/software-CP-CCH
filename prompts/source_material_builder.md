# Step 8 源程序鉴别材料

## 目标

依据已确认的代码选择清单，生成源程序鉴别材料草稿，并为最终 Word 源代码材料准备真实源码行段。

## 规则

- 程序和文档鉴别材料通常由前、后各连续 30 页组成；不足 60 页时提交全部。
- 除特定情况外，源程序每页不少于 50 行。
- 页眉或页首应保持软件全称和版本号一致。
- 代码应黑白、等宽字体、页码清晰；Markdown 草稿转 Word/PDF 时需复核。
- 代码必须来自真实项目，保留文件路径和行段用于追溯。

## 脚本

```bash
python tools/source_extract.py --project <项目路径> --selection 软件著作权申请资料/草稿/代码文件选择.json --software-name "<软件全称>" --version "<版本号>" --out-dir 软件著作权申请资料/草稿
```

## 自检

- 抽取文件都存在。
- 抽取行段连续且来自原文件。
- 前 30 页和后 30 页不重叠，除非全部不足 60 页。
- 页首软件名、版本号与申请表一致。
- 代码中无密钥、个人隐私、客户数据、未授权第三方标识。
- 代码没有被模型改写。

## Word 交付

正式交付阶段必须继续进入 `word_final_builder.md`，生成 `02 软件源代码 — <软件简称或英文名> <版本号>.docx`。

- Word 源码材料应使用等宽字体。
- 少于阈值时全量提交，超过阈值时按前后连续页/行提交。
- 源码行数必须来自真实源码统计；不得用模板虚报，不得为了页数制造无业务意义代码。
