# Step 6 申请表辅助信息

## 目标

整理中国版权保护中心在线申请表的填报辅助信息。不要伪装成官方表格；实际提交以官方系统当前页面为准。

## 必备字段

- 软件全称。
- 软件简称。
- 版本号。
- 著作权人 / 申请人。
- 开发完成日期。
- 首次发表日期或未发表。
- 权利取得方式：原始取得 / 受让 / 继承 / 承受等。
- 开发方式：独立开发 / 合作开发 / 委托开发 / 下达任务开发。
- 开发硬件环境。
- 运行硬件环境。
- 开发操作系统。
- 运行平台/操作系统。
- 开发工具。
- 运行支撑环境。
- 编程语言。
- 源程序量。
- 软件分类。
- 主要功能。
- 技术特点。
- AI 辅助或外部模型/API 使用说明（如适用）。

## 字段口径

- 软件全称和版本号必须由用户确认，并贯穿代码材料、手册、文件名、页眉。
- 开发工具填写 IDE 或编辑器；技术栈放在运行支撑环境或技术特点，不要混填。
- 源程序量来自真实项目统计，不要用模板虚报。
- 功能和技术特点来自 `业务理解.md` 和项目证据。
- 主要功能、技术特点、开发目的和软件分类必须逐项引用业务理解中的场景和证据；缺少实际业务对象或用户场景的字段只能标为待确认。
- 申请字段必须绑定已确认候选边界；不得把未纳入候选的软件子系统写进主要功能。
- AI 参与应如实、克制、可证据化，不写成宣传。

## 输出

先生成 `草稿/申请表信息.md`，格式建议：

```text
➤软件全称：...
➤版本号：...
➤著作权人：...
```

含“待用户确认”的字段不得进入正式资料。确认后运行：

```bash
python tools/confirm_stage.py --workdir 软件著作权申请资料 --stage application-fields --note "<用户确认内容>"
```

正式交付阶段必须继续进入 `word_final_builder.md`，生成申请表 `.docx`。申请表 Word 应基于用户/代理提供的模板或结构化表单生成，并做写后回读校验；不能只交 `申请表信息.md`。
