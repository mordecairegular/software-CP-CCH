# 交付自检

## 目标

在交付正式资料前做三轮检查，并把结果留档。自检不写入正式正文，但必须写入案件目录。最终交付应同时检查 Markdown 草稿和 Word 三件套。

## 三轮检查

### 第一轮：完整性

- 申请表辅助信息存在。
- 操作手册/说明书存在。
- 源程序鉴别材料存在。
- 代码提取清单存在。
- 权属与 AI 辅助审计存在。
- `正式资料/word/` 下存在申请表、源代码、操作说明书三份 `.docx`。
- 存在 `word_generation_report.md` 或等价 Word 生成/回读记录。
- 用户确认记录存在。

### 第二轮：真实性

- 代码能回溯到项目文件和行段。
- 手册功能能回溯到界面、源码、文档或用户说明。
- 截图是真实运行截图或可见占位。
- AI 辅助开发事实未被隐瞒或夸大。
- Word 源码材料来自真实源码，不以注释、水文或重复片段凑页数。
- Word 说明书截图来自真实软件运行状态，不包含无关 IDE、聊天窗口或桌面内容。

### 第三轮：一致性和格式

- 软件全称、简称、版本号一致。
- 日期、著作权人、开发方式一致。
- 页眉、文件名、申请表辅助字段一致。
- 源代码页数和行数规则符合当前材料口径。
- 申请表表格字段按标签正确落位；模板样本中的旧软件名、联系人、电话、邮箱不得残留。
- Word 文件可打开；如环境允许，渲染为 PNG/PDF 检查表格、图片和分页。
- 文案无明显 AI 味、营销口号、模板项目残留。

## 可用脚本

```bash
python tools/material_check.py --workdir 软件著作权申请资料
```

## 结果等级

- `PASS`：可供用户/代理人进入提交前人工复核。
- `WARN`：存在需要用户确认或人工补充的事项。
- `FAIL`：存在真实性、权属或核心材料缺失问题，不建议提交。

## 交付说明

向用户说明：

- 已生成哪些文件。
- 仍需用户上传或确认哪些官方系统字段/证明文件。
- 本结果不保证登记通过，以中国版权保护中心当前系统要求为准。
